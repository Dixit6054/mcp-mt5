# Specification: MT5 Headless Chart Management & Bash Telegram Monitoring

This document details the design for:
1. Optimizing headless MetaTrader 5 deployments so they only open the specific chart on which the Expert Advisor (EA) is attached.
2. Creating an hourly Bash monitoring script (`mt5_monitor.sh`) on the Linux VPS that tracks all running MT5 instances and reports their details (Login, Broker, EAs, RAM usage) via Telegram.

---

## 1. Headless Deployment Chart Management

### Objective
During headless deployment via the MCP tool `deploy_remote_instance`, ensure that:
* No extra/default charts are opened.
* The terminal only opens the chart for the specified symbol.
* The EA is automatically attached to this chart on startup.

### Proposed Approach
Instead of running a separate script to clean up Market Watch, we will perform chart management programmatically during the deployment workflow:

1. **Delete Existing Charts**:
   Before starting the systemd service for a new instance, the deployment script will delete any default charts cloned from the base prefix:
   ```bash
   rm -f "/home/ubuntu/<instance_name>/drive_c/Program Files/MetaTrader 5/Profiles/Charts/Default"/*.chr
   ```
2. **Configure Startup INI**:
   We will update `startup.ini` generated by the deployer to use MT5's native support for launching an EA on a specific chart:
   ```ini
   [Common]
   Login=<login>
   Password=<password>
   Server=<server>

   [Experts]
   AllowDllImport=1
   Enabled=1
   Expert=Experts\<EA_Name>.ex5
   Symbol=<Symbol>
   Period=H1
   # Optional parameters file
   # ExpertParameters=Presets\<Preset_Name>.set
   ```
   This ensures that when MT5 boots up, it opens exactly one chart (the one specified in `startup.ini`) and attaches the EA to it.

## 2. Bash Monitoring Script (`mt5_monitor.sh`)

### Objective
Create a lightweight, zero-dependency Bash script on the VPS that identifies running MT5 instances, parses their configuration files, and sends an hourly summary to Telegram.

### Script Logic
1. **Find MT5 PIDs**:
   ```bash
   pids=$(pgrep -f "terminal64.exe")
   ```
2. **Loop through PIDs and Extract Stats**:
   * **Memory Usage**: Extract Resident Set Size (RSS) in KB and convert to MB.
     ```bash
     mem_kb=$(grep -s VmRSS "/proc/$pid/status" | awk '{print $2}')
     mem_mb=$((mem_kb / 1024))
     ```
   * **Data Folder Path**: Locate the path from the mapped memory regions.
     ```bash
     mt5_path=$(grep -o -m 1 -a '/home/ubuntu/[^/]*\/drive_c\/Program Files\/MetaTrader 5' /proc/$pid/maps | head -n 1)
     ```
   * **Account & Broker**: Read and decode `Config/common.ini` (UTF-16 LE).
     ```bash
     common_utf8=$(iconv -f UTF-16LE -t UTF-8 "$mt5_path/Config/common.ini" 2>/dev/null)
     login=$(echo "$common_utf8" | grep -i "^Login=" | cut -d'=' -f2 | tr -d '\r')
     server=$(echo "$common_utf8" | grep -i "^Server=" | cut -d'=' -f2 | tr -d '\r')
     profile=$(echo "$common_utf8" | grep -i "^ProfileLast=" | cut -d'=' -f2 | tr -d '\r')
     ```
   * **Attached EAs**: Scan the active profile's chart configuration files (`chart*.chr`).
     ```bash
     # Convert chart file to UTF-8 and extract expert name and symbol
     chr_utf8=$(iconv -f UTF-16LE -t UTF-8 "$chr_file")
     if echo "$chr_utf8" | grep -q "<expert>"; then
         symbol=$(echo "$chr_utf8" | grep -i "^symbol=" | cut -d'=' -f2 | tr -d '\r')
         ea_name=$(echo "$chr_utf8" | awk '/<expert>/{flag=1;next}/<\/expert>/{flag=0} flag' | grep -i "^name=" | cut -d'=' -f2 | tr -d '\r')
     fi
     ```
3. **Format and Send Message**:
   * Construct a markdown message.
   * Send to Telegram using `curl`:
     ```bash
     curl -s -X POST "https://api.telegram.org/bot8323264961:AAFg5q-dzTCwrWCeYhAOqJNOco8trHY3eJw/sendMessage" \
          -d "chat_id=-1003908299290" \
          -d "text=$MESSAGE" \
          -d "parse_mode=Markdown"
     ```
